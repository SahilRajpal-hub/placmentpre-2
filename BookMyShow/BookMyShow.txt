Improvement 1 — Concurrency
    Problem:
        Multiple users booking same seat simultaneously
    Solutions
    Option 1 — DB Row Locking
        SELECT ... FOR UPDATE
    Option 2 — Distributed Lock
        Redis/Zookeeper.
    Option 3 — Optimistic Locking
        Version-based updates.

    say : Seat booking is a classic high-contention concurrency problem.

Improvement 2 — Lock Expiry
    If user abandons payment: seats should auto-release
    Approaches:
        scheduler
        TTL cache
        Redis expiry
