----- Biggest Design Decision
SAY : “The key challenge is modeling piece movement cleanly without creating large conditional logic.”

BAD:
if(piece == KING) ...
else if(piece == QUEEN) ...

Instead:
    polymorphism
    strategy-based movement

------ suggest improvements
Improvement-1 : Strategy Pattern in Movement rules.
Alternative design:

interface MoveStrategy {
    boolean validate(...);
}
Used by pieces.


Improvement-2 : Command Pattern
Move itself becomes command.
Useful for:
    undo
    replay
    persistence



Improvement-3 : State Pattern
Game states:
    active
    paused
    ended



Improvement-4 : Observer Pattern
UI updates:
    onMove()
    onCheck()
    onCheckmate()

Decouples backend/UI.
